# Code of Conduct

## Our Standard

This project is built around **trust, clarity, and professionalism**.

We expect everyone contributing — whether code, ideas, or feedback — to:

* Communicate clearly and respectfully
* Stay focused on solving real problems
* Avoid unnecessary complexity or over-engineering
* Take responsibility for the quality of their work

## What We Don’t Accept

* Disrespectful, aggressive, or dismissive behaviour
* Repeated low-effort or careless contributions
* Introducing unnecessary complexity without justification
* Ignoring established patterns or project structure

## How We Work

* **Keep things simple and aligned** — follow existing patterns before introducing new ones
* **Be precise** — avoid vague changes or guesswork implementations
* **Stay in scope** — don’t refactor unrelated areas without agreement
* **Communicate clearly** — especially when something is uncertain

## Reporting Issues

If you experience or observe behaviour that goes against this standard, report it to:

**[constructivesolutionsibiza@gmail.com](mailto:constructivesolutionsibiza@gmail.com)**

All reports will be handled seriously and fairly.

## Enforcement

We reserve the right to:

* Reject contributions that don’t meet standards
* Request changes before accepting work
* Remove access if behaviour repeatedly goes against the project

---

This is not about control — it’s about maintaining a high standard so the project stays strong as it grows.
